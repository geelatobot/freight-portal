# 代码审查清单 (Code Review Checklist)

## 审查原则
- **安全性**: 无敏感信息泄露
- **可读性**: 代码易于理解
- **可测试性**: 易于测试
- **性能**: 无性能隐患

## 审查项目

### 1. 安全性检查
- [ ] 无硬编码密码/API Key
- [ ] 输入参数已验证
- [ ] SQL注入防护（使用Prisma ORM）
- [ ] XSS防护
- [ ] CSRF防护

### 2. 代码质量
- [ ] 符合TypeScript类型规范
- [ ] 函数/变量命名清晰
- [ ] 注释完整（复杂逻辑）
- [ ] 无console.log（使用logger）
- [ ] 错误处理完善

### 3. 测试覆盖
- [ ] 单元测试覆盖核心逻辑
- [ ] 集成测试覆盖API接口
- [ ] 测试用例包含边界条件
- [ ] 测试覆盖率 > 80%

### 4. 性能优化
- [ ] 数据库查询有索引
- [ ] 无N+1查询问题
- [ ] 缓存使用合理
- [ ] 异步处理正确

### 5. 文档
- [ ] API文档更新
- [ ] 代码注释清晰
- [ ] 变更日志记录

## 审查流程

1. **提交PR** → 自动运行CI测试
2. **代码审查** → 至少1人批准
3. **测试通过** → 覆盖率达标
4. **合并代码** → 自动部署到测试环境
5. **验收测试** → 通过后再部署生产
